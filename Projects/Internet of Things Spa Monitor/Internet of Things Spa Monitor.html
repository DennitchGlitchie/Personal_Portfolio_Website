Internet of Things: Spa Monitor

(Web Hosted Temperature Control, PH, and Video Monitor of Outdoor Spa)

Summary:

This spa monitoring system is focused around a raspberry pi that is local to a hottub and has been an exercise in Interface and Network Technologies. The Raspberry Pi is able to measure temperature (USB), PH (I2C), and video (Pi Camera Ribbon) stored on an SQLite Database while controlling output status to an Actuator LED (GPIO) and LCD (SPI). The Raspberry Pi 2 uses a wifi dongle to connect to the local wifi network. I have implemented a web server through flask that allows a user on the network to see the measurement data from the Hot Tub and provide the ability for the user to change the status of the actuator LED.

Technical Skills Utilized:

  • Sensor Interface Technologies
  • Internet of Things Connectivity

Prototype:

Environment Input:

Temperature, PH, and Light will be detected by the Temperature Probe, PH Probe, and Camera (respectively). This data is appropriately formatted to be stored locally on a SQLite database on the Raspberry Pi. As the user refreshes and interacts with the web server hosted by Flask, the data is uploaded appropriately to the web application.

User Input:

As the user faces the UI of the web application, they are able to toggle the status of the actuator. This is handled by flask and instructs the Raspberry Pi through the python language to change the status of the actuator via GPIO. The status of the actuator is updated on the web interface as well as on the local LCD.

Describing Interface Technologies:

  • Atlas Scientific Temperature Probe through USB uses 12Mbit/s standard USB protocol.
  • Atlas Scientific PH Probe through I2C uses 3.3V Logic, 100kHz Clock, and a default slave address of 0x63.
  • Nokia 5110 LCD through SPI uses 3.3V Logic, 7.8MHz Clock.
  • Off-Shelf LED through Raspberry Pi GPIO
  • Raspberry Pi V2 through standard Raspberry Pi Camera Interface.

Describing Network Technologies:

Flask is used to host a web server and web page for User Interface.

Understanding the Networking Layers:

  • Application Layer: HTTP and Flask
  • Transport Layer: TCP Stack utilized by Flask
  • Network Layer: IPv4 Protocol
  • Local IP address: 192.168.0.32:5000
  • Using port 5000

Website Wire Framework:

I have a very simple and straightforward web framework. When a person on the local network give the URL: 192.168.0.32:5000, the homepage comes up which allows the user to see a recent Temperature and PH reading. The home page will also give the status of the LED actuator by reading the GPIO state as an input. The home page offers a button which will toggle the state of the actuator. I have enabled this feature by linking to ‘/cakes’ on-click. This page has no template and essentially works behind the scenes. It detects the GPIO status and flips it. ‘/cakes’ immediately routes back to the home page where the LCD and the LED status is appropriatel§y changed and displayed.

Atlas Scientific's ENIAC Hosts and Logs Data (Early Development):

ENIAC is a program offered by Atlas Scientific that is installed on the Raspberry Pi to easily communicate with sensors to host and tabulate recorded data. The temperature and the PH probes must be connected to the Raspberry pi's USB interface through a EZO BNC to USB circuit.

Discussion of waterproof housing:

When all electrical components are working correctly, the system will be enclosed in a waterproof housing that can sustain humidity and splashing from the Spa as well as rain and heat from the elements.
